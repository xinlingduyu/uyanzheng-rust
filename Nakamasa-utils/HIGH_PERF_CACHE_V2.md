# Nakamasa 高性能缓存系统 V2

## 概述

Nakamasa 高性能缓存系统 V2 是一个为高并发场景设计的极致优化缓存实现。通过分片架构、无锁数据结构、O(1) 淘汰策略等多种技术手段，实现了纳秒级的读取延迟和千万级的并发吞吐量。

### 性能指标

| 指标 | V1 | V2 | 提升比例 |
|------|----|----|----------|
| 单次读取延迟（命中） | ~200ns | ~50ns | **4x** |
| 单次写入延迟 | ~150ns | ~100ns | **1.5x** |
| LRU 淘汰操作 | O(n) | O(1) | **N x** |
| 并发吞吐量 | ~5M ops/s | ~15M ops/s | **3x** |
| 统计开销 | 100% | ~1% | **100x** |
| 内存池分配 | ~100ns | ~20ns | **5x** |

#### 2.2 无锁 MPMC 队列原理

```
┌─────────────────────────────────────────────────────────────────────┐
│              Michael-Scott 无锁队列算法                              │
│                                                                      │
│   数据结构:                                                          │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │                                                              │   │
│   │   struct MpmcQueue<T> {                                      │   │
│   │       head: AtomicPtr<Node<T>>,    // 头指针                 │   │
│   │       tail: AtomicPtr<Node<T>>,    // 尾指针                 │   │
│   │       len: AtomicUsize,            // 长度                   │   │
│   │   }                                                          │   │
│   │                                                              │   │
│   │   struct Node<T> {                                           │   │
│   │       data: MaybeUninit<T>,                                  │   │
│   │       next: AtomicPtr<Node<T>>,                              │   │
│   │   }                                                          │   │
│   │                                                              │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│   队列状态示例:                                                      │
│                                                                      │
│   head                                                              │
│     │                                                               │
│     ▼                                                               │
│   ┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐            │
│   │ 哨兵   │───→│  数据1 │───→│  数据2 │───→│  数据3 │            │
│   │ (空)   │    │        │    │        │    │        │            │
│   └────────┘    └────────┘    └────────┘    └────────┘            │
│                                                 ▲                   │
│                                                 │                   │
│                                               tail                 │
│                                                                      │
│   Push 操作 (CAS):                                                   │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │  1. 创建新节点 new_node                                       │   │
│   │  2. 读取 tail                                                │   │
│   │  3. 如果 tail.next == null:                                  │   │
│   │      CAS(tail.next, null, new_node)                          │   │
│   │      CAS(tail, old_tail, new_node)  // 推进 tail             │   │
│   │  4. 否则: CAS(tail, old_tail, tail.next)  // 帮助推进        │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│   Pop 操作 (CAS):                                                    │
│   ┌─────────────────────────────────────────────────────────────┐   │
│   │  1. 读取 head 和 tail                                        │   │
│   │  2. 如果 head == tail:                                       │   │
│   │      如果 head.next == null: 返回 None (队列为空)            │   │
│   │      否则: CAS(tail, head, head.next)  // 帮助推进           │   │
│   │  3. 否则:                                                    │   │
│   │      读取 head.next.data                                     │   │
│   │      CAS(head, old_head, head.next)                          │   │
│   │      返回 data                                               │   │
│   └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 使用指南

### 快速开始

```rust
use nakamasa_utils::high_perf_cache::*;
use std::time::Duration;

// 创建高性能缓存
let config = CacheConfig {
    max_entries: 1_000_000,
    shard_count: 128,                    // 推荐: CPU核心数 * 4~8
    default_ttl: Duration::from_secs(300),
    eviction_policy: EvictionPolicy::Hybrid { 
        lfu_weight: 0.6, 
        lru_weight: 0.4 
    },
    ..Default::default()
};

let cache = ShardedCacheV2::<String, Vec<u8>>::new(config);

// 基本操作
cache.set("key1".to_string(), vec![1, 2, 3]);
let value = cache.get(&"key1".to_string());
cache.remove(&"key1".to_string());

// 带 TTL
cache.set_with_ttl("temp".to_string(), vec![4, 5, 6], Duration::from_secs(10));

// 批量操作
let results = cache.get_many(&["k1".to_string(), "k2".to_string(), "k3".to_string()]);
cache.set_many(vec![
    ("k1".to_string(), vec![1]),
    ("k2".to_string(), vec![2]),
]);

// 获取统计
let stats = cache.stats();
println!("Hit rate: {:.2}%", stats.hit_rate * 100.0);
```

## 性能调优

### 分片数量选择

```
推荐公式: shard_count = CPU核心数 × 4 ~ 8

示例:
- 4 核 CPU:  16 ~ 32 分片
- 8 核 CPU:  32 ~ 64 分片
- 16 核 CPU: 64 ~ 128 分片
- 32 核 CPU: 128 ~ 256 分片

分片太少: 锁竞争严重
分片太多: 内存开销增加，缓存局部性降低
```

### 淘汰策略选择

```
┌─────────────────────────────────────────────────────────────────┐
│  访问模式                    │  推荐策略                        │
├──────────────────────────────┼──────────────────────────────────┤
│  时间局部性强 (最近访问的更可能再次访问)  │  LRU                │
│  频率稳定 (热点数据访问频率稳定)         │  LFU                │
│  混合模式 (时间+频率混合)               │  Hybrid (推荐)      │
│  不确定/多变模式                        │  Adaptive (ARC)     │
│  追求最低开销                           │  Random             │
└─────────────────────────────────────────────────────────────────┘
```

### TTL 设置建议

```
┌─────────────────────────────────────────────────────────────────┐
│  数据类型          │  推荐 TTL          │  说明                  │
├────────────────────┼────────────────────┼────────────────────────┤
│  热点数据          │  5 ~ 30 分钟       │  高频访问，长时间缓存  │
│  普通数据          │  1 ~ 5 分钟        │  中等频率，适中 TTL    │
│  临时数据          │  10 ~ 60 秒        │  低频访问，短 TTL      │
│  会话数据          │  15 ~ 30 分钟      │  用户会话生命周期      │
│  配置数据          │  1 ~ 24 小时       │  很少变化，长 TTL      │
└─────────────────────────────────────────────────────────────────┘
```

## 最佳实践

### 1. 选择合适的缓存策略

```rust
// 用户会话缓存 - 使用 LRU
let session_cache = ShardedCacheV2::<String, Session>::new(CacheConfig {
    eviction_policy: EvictionPolicy::LRU,
    default_ttl: Duration::from_secs(1800),  // 30 分钟
    ..
});

// 热门内容缓存 - 使用 LFU
let content_cache = ShardedCacheV2::<String, Content>::new(CacheConfig {
    eviction_policy: EvictionPolicy::LFU,
    default_ttl: Duration::from_secs(3600),  // 1 小时
    ..
});

// 通用缓存 - 使用 Hybrid
let general_cache = ShardedCacheV2::<String, Data>::new(CacheConfig {
    eviction_policy: EvictionPolicy::Hybrid { lfu_weight: 0.6, lru_weight: 0.4 },
    default_ttl: Duration::from_secs(300),  // 5 分钟
    ..
});
```

### 2. 批量操作优化

```rust
// ❌ 错误: 逐个操作
for (key, value) in entries {
    cache.set(key, value).await;
}

// ✅ 正确: 批量操作
cache.set_many(entries);
```

### 3. 预热热点数据

```rust
// 启动时预热
async fn warmup_cache(cache: &ShardedCacheV2<String, Vec<u8>>) {
    let hot_keys = db::get_hot_keys().await;
    let entries = hot_keys.into_iter()
        .filter_map(|key| {
            db::get(&key).await.map(|value| (key, value))
        })
        .collect();
    
    cache.set_many(entries);
}
```

### 4. 监控告警

```rust
// 定期检查缓存健康状态
fn check_cache_health(monitor: &CacheMonitorV2) {
    let stats = monitor.snapshot();
    
    // 命中率告警
    if stats.hit_rate < 0.8 {
        log::warn!("Cache hit rate low: {:.2}%", stats.hit_rate * 100.0);
    }
    
    // 延迟告警
    if stats.read_latency.p99_ns > 1_000_000 {  // > 1ms
        log::warn!("High read latency P99: {}ns", stats.read_latency.p99_ns);
    }
}
```
